마크업(Mark up)

  • SEMANTIC LAYOUT

    Doctype

    DOCTYPE은 html5 버전을 선언. <!DOCTYPE html>

    META Content

    iPhone X 등의 Notch를 가지고 있는 기종의 스크린사이즈 조절을 위해 "viewport-fit=cover" 추가. <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0, user-scalable=no, viewport-fit=cover">

    HEADER

    header는 html5 semantic code를 사용하여 header 태그를 사용하며, 항상 클래스 header를 포함.
    <!-- header 시작 -->
    <header class="header">
    </header>
    <!-- header 끝 -->     단, 풀팝업(FULL POP-UP) 및 팝업(POP-UP)의 경우 div 태그에 header 클래스를 사용하여 페이지 header와 차별 <!-- header 시작 -->
    <div class="header">
    </div>
    <!-- header 끝 -->

    NAV

    상단 공통 네비게이션 메뉴는 header 태그 안에 nav로 선언.

    뒤로가기 버튼, 페이지 타이틀 스타일

    <!-- header 시작 -->
    <header class="header">
      <!-- nav 시작 -->
      <nav>
        <a href="javascript:void(0)" class="backBtn">뒤로가기</a>
        <h1 class="headerTitle"><a href="javascript:void(0)">e금리우대적금</a></h1>
      </nav>
      <!-- nav 끝 -->
    </header>
    <!-- header 끝 -->

    페이지 타이틀, 닫기 (팝업) 스타일

    <!-- header 시작 -->
    <div class="header">
      <!-- nav 시작 -->
      <nav>
        <h1 class="headerTitle"><a href="javascript:void(0)">e금리우대적금</a></h1>
        <a href="javascript:void(0)" class="closeBtt">닫기</a>
      </nav>
      <!-- nav 끝 -->
    </div>
    <!-- header 끝 -->

    전체메뉴,설정, 페이지 타이틀과 로그인 스타일

    <!-- header 시작 -->
    <header class="header">
      <nav>
        <ul>
          <li><a href="javascript:void(0)" class="headerNav navBtn">전체메뉴</a></li>
          <li><a href="javascript:void(0)" class="headerNav settingBtn">설정</a></li>
        </ul>
      <h1 class="headerTitle"><a href="javascript:void(0)">마이올원</a></h1>
      <a href="javascript:void(0)" class="headerNav loginBtn">로그인</a>
      </nav>
    </header>
    <!-- header 끝 -->

    CONTENTS

    모든 컨텐츠의 시작은 data-role="content" 값을 가진 div태그로 시작
    컨텐츠는 section태그로 시작하며, 컨텐츠를 분리할 경우 section태그로 구분.
    section 태그가 가지는 클래스명에 따라 공통 요소 블럭 재활용 가능.
    section 태그 자체가 가지는 양쪽 여백 15px을 배제하여야 할 경우
    클래스 'fullWidth'추가시 100% 넓이의 컨테이너로 활용 가능.
    </header>
    <!-- header 끝 -->

    <!-- content 시작 -->
    <div data-role="content">
      <!-- breadcrumb 시작 -->
      <section class="breadcrumb columnSet">
        ....
      </section>
      <!-- breadcrumb 끝 -->

      <!-- form 시작 -->
      <section>
        <form>
        <fieldset class="formWrap">
          .....
        </fieldset>
        </form>
      </section>
      <!-- form 끝 -->
    </div>
    <!-- content 끝 -->

    <!-- footer 시작 -->
    <footer>

텍스트 Global Class

기본 공통 스타일 가이드에서는 벗어나 강제로 스타일 변경이 있어야 할 경우 아래의 클래스를 추가하면 특정 요소 스타일 변경이 가능

uppercase
영문 전체를 대문자로 변경
lowercase
영문 전체를 소문자로 변경
capitalize
영문 제일 첫글자만 대문자로 변경
italic
텍스트를 이탤릭 스타일로 변경
fontBold
텍스트를 볼드 스타일로 변경
fontNormal
텍스트를 볼드체가 아닌 노멀 스타일로 변경
centeredTxt
텍스트를 가운데 정렬
leftTxt
텍스트를 왼쪽 정렬
rightTxt
텍스트를 오른쪽 정렬
valign-middle
텍스트 세로 가운데 정렬
valign-top
텍스트를 세로 위 정렬
valign-bottom
텍스트를 세로 아래 정렬
greenOn
텍스트를 하이라이트 그린 컬러로 변경

Column Set

페이지 특정 요소가 페이지 전체 넓이 또는 BLOCK 넓이를 기준으로 요소의 사이즈에 따라 배분되어 1렬로 디스플레이가 필요한 경우 사용.
모든 요소에 사용가능. 단, 'columnSet'이라고 하는 클래스는 요소들의 부모요소 즉, 디스플레이 되어질 요소들을 감싸는 wrapper에 선언


** Column Set요소 CLASS

columnSet
요소들을 감싸는 부모 'div' 또는 'ul'요소에 선언
withGutter
각각의 요소들의 오른쪽에 여백이 필요할 경우 클래스 추가
colItem
각각의 컬럼의 넓이가 동일하게 디스플레이가 필요한 경우 컬럼 'li' 또는 특정 자식 요소에 클래스 추가
rowSpan1
넓이가 10% // 자식 요소에 상이한 넓이값을 주고 싶을 경우
rowSpan2
넓이가 16.6666% // 자식 요소에 상이한 넓이값을 주고 싶을 경우
rowSpan3
넓이가 25% // 자식 요소에 상이한 넓이값을 주고 싶을 경우
rowSpan4
넓이가 33.3333% // 자식 요소에 상이한 넓이값을 주고 싶을 경우
rowSpan6
넓이가 50% // 자식 요소에 상이한 넓이값을 주고 싶을 경우
  <!-- columnSet 시작 -->
  <ul class="columnSet withGutter">
    <li class="colItem">1</li>
    <li class="colItem">2</li>
    <li class="colItem">3</li>
    <li class="colItem">4</li>
    <li class="colItem">5</li>
  </ul>
  <!-- columnSet 끝 -->

<fieldset class="formWrap">

Breadcrumb

상품 가입이나 회원가입등의 스텝별 위치 확인이 필요한 경우 페이지 상단에 붙는 breadcrumb


** Breadcrumb요소 CLASS

breadcrumb
breadcrumb wrapper
step
breadcrumb order list
is-off
아직 진행되지 않은 스텝
is-on
현재 진행중인 스텝
is-done
완료한 스텝 is-off 클래스와 동시에 선언되어야 한다.
  <!-- breadcrumb 시작 -->
  <section class="breadcrumb columnSet">
    <h2>상품가입 약관 및 동의서</h2>
    <ol class="step columnSet">
      <li class="is-off is-done">1</li>
      <li class="is-on">2</li>
      <li class="is-off">3</li>
    </ol>
  </section>
  <!-- breadcrumb 끝 -->

MODAL

'modalWrap' 기본 팝업 wrapper이며 'dimmed' 클래스가 같이 있어야 배경색이 적용
팝업은 풀팝업(Full-Popup)과 기본팝업 두가지로 구분.
기본팝업의 클래스는 modalDefault이고, 풀팝업의 경우 modalFull로 클래스를 사용.
팝업은 header, modal-contents, columnSet 으로 구분
header는 상단 타이틀과 닫기 버튼으로 구성
modal-contents는 컨텐츠로 구성
columnSet은 하단에 버튼셋이 필요한 경우 content 블럭 아래에 위치
<div class="modalWrap">

  </div>
  <div data-role="content" class="modal-contents">

CSS(Cascading Style Sheets)

  • Single-line 코드로 작성한다.
  • !important 사용 자제한다.
  • iframe으로 불어오는 페이지가 많이 있어서 전체 선택자(*) 사용을 자제
  • 색상 스타일 선언시 축약형을 사용한다. .txt{color:#333333;} (x)
    .txt{color:#333;} (o)

  • z-index 값 적용 가이드

    • z-index 수치 간격은 100단위로 적용
    • 팝업레이어의 경우는 1000부터 시작
    • 최고 수치는 9999로 고정
    • 필요시 z-index를 재정의
  • 자바스크립트에서 주로 이벤트 trigging을 위해 사용되는 ID에는 스타일 적용하지 않는다.

  • class명에 html tag는 사용하지 않는다.

    div.header (X)
